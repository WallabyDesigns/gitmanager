Queue Behavior

When the deploy queue is enabled, auto-deploys and webhooks are queued and processed sequentially. Manual deploys run immediately and take priority.

  • Manual deploys bypass the queue and clear queued deploys for the same project
  • Only one deploy runs at a time per project
  • Queued items can be processed individually, cancelled, reordered, or force-cancelled if stuck

Enable / Disable

Toggle the queue via GWM_DEPLOY_QUEUE_ENABLED in .env.

Queue Processor

The scheduler runs deployments:process-queue every minute to process queued items.

If the queue is not advancing, visit Scheduler Settings to verify the heartbeat, run the scheduler manually, or install the cron line.
