Git Project Manager

Self-hosted Git deployment manager with health checks, rollbacks, preview builds, and Dependabot automation.

Highlights

  • Deploy and rollback from a single dashboard.
  • Auto-deploy via schedule or GitHub webhooks.

Quick Start

  1. Clone the repository to your server.
  2. Configure environment variables in .env.
  3. Install dependencies and run migrations.
  4. Log in and add your first project.
composer install
php artisan migrate
npm install
npm run build

Installation

  • PHP 8.2+ with required extensions.
  • Composer 2.
  • Node.js 18+ (or 20/22) for builds.
  • Git CLI available to the web user.
  • Queue worker for webhook deploys.

Recommended: run on the same host where your projects live so deploys can write to local paths.

Configuration

Key settings in .env:

  • GITHUB_TOKEN for private repos + Dependabot.
  • GITHUB_WEBHOOK_SECRET for push-triggered deploys.
  • GPM_GIT_BINARY, GPM_COMPOSER_BINARY, GPM_NPM_BINARY for custom CLI paths.
  • GPM_PHP_BINARY / GPM_PHP_PATH for PHP CLI selection.
  • GPM_PROCESS_PATH to prepend PATH (Node, PHP, etc).
  • GPM_SELF_UPDATE_ENABLED to toggle nightly self updates.
  • GPM_PREVIEW_PATH and GPM_PREVIEW_BASE_URL for preview builds.

Projects & Deployments

Create a project by providing:

  • Repo URL (HTTPS, SSH, or owner/repo shorthand).
  • Local path (existing or auto-created).
  • Branch (default branch to deploy).

Deploy actions:

  • Deploy / Force Deploy (with optional clean).
  • Rollback to last successful deploy.
  • Dependency actions (composer/npm).

You can preserve custom files via Exclude Paths. Each entry is one path per line (supports globs).

Health Checks

Set a health URL (default: /up for Laravel 11). Health status refreshes every minute on list and detail views.

Preview Builds

Generate previews for any commit without touching production. Configure GPM_PREVIEW_PATH and optionally GPM_PREVIEW_BASE_URL.

Example preview path:

/previews/my-project/abc1234

Security & Dependabot

  • Syncs Dependabot alerts hourly.
  • Auto-merges Dependabot PRs (configurable).
  • Security tab shows open vs resolved issues.

Make sure GITHUB_TOKEN has repo and security_events scopes.

GitHub Webhooks

Set your GitHub webhook to:

POST /webhooks/github

Use the same GITHUB_WEBHOOK_SECRET value in GitHub and your .env.

Webhook deploys are queued — run a worker:

php artisan queue:work

Self Update

The app can self-update nightly (02:30 server time) and also includes a manual Update App page.

  • Status badge shows if a new version is available.
  • Updates preserve .htaccess and optional paths.
  • NPM install is skipped when package files have not changed.

Use Cases

  • Small agencies managing multiple client sites.
  • Solo developers with multiple side projects.
  • Internal teams that need simple Git deploys.
  • Preview builds for QA or staging review.

Troubleshooting

  • Vite manifest missing: run npm run build or fix permissions on public/build.
  • Git auth failed: verify GITHUB_TOKEN and PATH for git.
  • Health checks fail: confirm URL and HTTPS access.
  • Deploy stuck: check queue worker and job logs.
