Create Workflow Rules

Workflows are rule-based notifications. Each rule listens for an action and outcome, then fires an email or webhook.

  • Choose an action (deploy, rollback, dependency update, preview build, composer/npm, custom commands).
  • Choose an outcome (success or failed).
  • Select a channel (email or webhook).

Use the tabs on the Workflows page to switch between the current rules list and the create form.

Webhook Payload

Webhooks receive a JSON payload describing the action.

{
  "event": "deploy.deploy_success",
  "event_key": "deploy_success",
  "event_standard": "deploy.success",
  "timestamp": "2026-03-18 12:00:00",
  "project": {
    "id": 1,
    "name": "Example",
    "repo": "https://github.com/...",
    "branch": "main",
    "path": "/var/www/example"
  },
  "deployment": {
    "id": 42,
    "status": "success",
    "action": "deploy",
    "from": "abc123",
    "to": "def456",
    "started_at": "2026-03-18 11:59:00",
          "finished_at": "2026-03-18 12:00:00"
        },
        "workflow": {
          "id": 12,
          "name": "Deploy success webhook",
          "channel": "webhook",
          "action": "deploy",
          "status": "success"
        }
}

If a webhook secret is configured, requests include an X-GWM-Signature HMAC header.
